# OneTab Plus - Chrome标签管理器 (v1.4.6)

这是一个Chrome浏览器扩展程序，用于高效管理和组织浏览器标签页。现已发布稳定的 1.4.6 版本。

## 功能特点

- 一键保存所有打开的标签页
- 将多个标签页转换为列表形式保存，减少内存占用
- 支持标签组管理和分类
- 支持标签页的导入/导出
- 支持云端同步，多设备访问
- 支持实时双向同步，多设备数据实时更新
- 支持标签拖拽操作的实时同步
- 使用 JSONB 格式存储数据，提高性能
- 支持标签页的恢复和删除
- 支持标签组的命名和重命名
- 支持标签页的搜索功能
- 自动过滤 Chrome 内部页面（chrome://）和扩展页面（chrome-extension://）
- 可配置是否允许保存重复的标签页
- 支持快捷键操作：
  - Alt+Shift+S: 保存所有标签页
  - Alt+S: 保存当前标签页
  - Ctrl+Shift+S (Mac: Command+Shift+S): 打开扩展弹窗

## 技术栈

- TypeScript
- React
- Redux Toolkit
- Chrome Extension API
- Vite
- TailwindCSS
- Supabase (后端服务和认证)

## 开发环境设置

1. 克隆仓库
```bash
git clone https://github.com/yourusername/chrome-plugin-one-tab.git
cd chrome-plugin-one-tab
```

2. 安装依赖
```bash
npm install
```

3. 开发模式运行
```bash
npm run dev
```

4. 构建生产版本
```bash
npm run build
```

## 在Chrome中安装开发版本

1. 打开Chrome浏览器，进入扩展程序页面 (chrome://extensions/)
2. 开启"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择项目的 `dist` 目录

## 使用说明

1. 点击工具栏中的扩展图标，将当前所有标签页保存到列表中
2. 在扩展管理页面中可以：
   - 查看所有保存的标签组
   - 恢复单个标签或整组标签
   - 对标签组进行重命名
   - 删除不需要的标签或标签组
   - 搜索已保存的标签页
   - 导入/导出标签数据
   - 访问设置页面进行个性化配置

## 最近更新 (v1.4.3)

### v1.4.3 更新

- **界面优化**：
  - 添加双栏布局，提高信息密度，更有效利用屏幕空间
  - 添加布局切换按钮，允许用户自由切换单栏/双栏布局
  - 优化标签组卡片设计，添加圆角和阴影效果

- **搜索功能增强**：
  - 优化搜索结果显示，不再按标签组分组显示，而是直接显示标签列表
  - 添加搜索结果页面的“恢复全部”功能，允许用户一次性打开所有搜索到的标签页

- **标签组管理增强**：
  - 添加了标签组重命名按钮，使重命名操作更直观
  - 锁定的标签组不能重命名，需要先解锁

### v1.4.0 更新

- **界面优化**：
  - 合并右上角的下拉菜单，避免重复显示
  - 添加失去焦点自动关闭下拉菜单功能
  - 修复设置页面返回按钮的路径问题

- **错误修复**：
  - 修复了 Service Worker 加载失败的问题
  - 修复了设置页面返回按钮导致 404 错误的问题
  - 修复了路径不一致导致的各种问题

### v1.2.0 更新

- **云端同步功能**：
  - 支持用户注册和登录
  - 自动同步标签组和设置到云端
  - 多设备间数据同步
  - 智能合并算法处理冲突

- **数据压缩功能**：
  - 使用 LZ-string 压缩数据，减少网络使用
  - 压缩率通常可达 50-70%
  - 显著提高同步速度
  - 压缩统计信息显示

- **同步策略设置**：
  - 可选择冲突解决策略（最新版本、本地版本、云端版本、询问用户）
  - 可选择删除策略（所有设备删除、仅本地删除）
  - 自定义同步频率

### v1.1.0 更新

- **界面优化**：
  - 采用更紧凑的标签显示方式，类似 OneTab 的展示方式
  - 移除了深色模式功能，使界面更简洁
  - 不再显示每个链接的具体 URL，只显示标题

### v1.0.0 初始版本

- **搜索功能优化**：
  - 将"搜索标签组"改为"搜索标签"
  - 搜索结果现在只显示匹配的标签，而不是显示整个标签组
  - 每个搜索结果显示标签所属的标签组

- **重复标签页处理**：
  - 添加了"允许保存重复的标签页"设置选项
  - 默认情况下，保存标签页时会过滤掉重复的 URL，只保留最新的一个

### 问题修复

- 修复了同步失败时的错误处理
- 修复了数据同步时可能丢失本地数据的问题
- 修复了 `null value in column "updated_at"` 错误
- 修复了在弹出窗口中恢复标签页后，标签页没有从列表中删除的问题
- 改进了标签页恢复和删除的错误处理

# TabVault Pro Manual Test Checklist

目标：在每次发布前用 10 到 15 分钟跑完关键闭环，确认“保存 -> 找回 -> 恢复 -> 手动同步”没有回归。

## 发布前先检查

- `package.json`、`manifest.json`、界面页脚版本一致
- README 中的“当前版本”与本次版本一致
- 执行 `pnpm type-check`
- 执行 `pnpm lint`
- 执行 `pnpm build`

## 10 个核心手测项

1. 保存当前窗口
- 打开 4 到 6 个常规网页
- 点击“保存会话”
- 预期：生成一个新会话；原窗口标签被收起；提示文案说明已保存为会话

2. 恢复整个会话
- 从列表中恢复一个未锁定会话
- 预期：在新窗口打开；提示包含标签页数量；原会话从列表移除

3. 恢复单个标签
- 从一个会话中点击单个标签恢复
- 预期：目标页面被打开；该标签从原会话中移除；若会话为空则自动删除

4. Pinned 关闭时保存
- 关闭“保存固定标签页”
- 准备至少 1 个 pinned tab 和 2 个普通 tab
- 保存会话
- 预期：pinned tab 不进入会话，也不会被关闭

5. Pinned 开启时保存
- 打开“保存固定标签页”
- 再次保存含 pinned tab 的窗口
- 预期：会话中包含 pinned tab；恢复时 pinned 状态保留

6. OneTab 导入
- 导入一个有效的 OneTab 文本文件
- 预期：导入成功；列表里新增可搜索、可恢复的会话

7. OneTab 导出
- 导出当前数据为 OneTab 文本
- 预期：成功下载文本文件；内容可读且非空

8. 登录 / 登出
- 完成登录，再执行登出
- 预期：登录后出现同步按钮和最后同步信息；登出后同步入口消失

9. 手动上传
- 登录后点击上传
- 分别验证“覆盖模式”和“合并模式”至少一种
- 预期：操作前说明清楚；成功后 toast 与实际行为一致

10. 手动下载
- 登录后点击下载
- 分别验证“覆盖模式”和“合并模式”至少一种
- 预期：操作前说明清楚；成功后 toast 与实际行为一致；无本地数据时会直接恢复云端数据

## 搜索专项

- 搜索会话名能返回整组会话
- 搜索备注、标签标题或 URL 能先看到匹配会话，再看到具体标签
- 域名、固定标签、保存时间筛选能正常收敛结果

## 会话元数据专项

- 收藏一个会话后，主列表排序会优先显示该会话
- 给会话添加备注后，刷新页面仍然保留
- 从列表、搜索、最近保存恢复会话后，最近恢复区会记录最近一次来源

## 失败即阻塞发布

- 无法保存当前窗口
- 无法恢复整个会话
- 恢复后不是新窗口
- 同步文案与真实行为不一致
- README / manifest / 页脚版本不一致

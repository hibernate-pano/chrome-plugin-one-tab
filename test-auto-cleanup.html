🎯 功能概述

新功能：在用户拖拽标签页到其他标签组时，如果源标签组在拖拽操作完成后变为空（即不包含任何标签页），则自动删除该空的标签组。

触发条件：跨组拖拽 + 源标签组变空 + 标签组未锁定

✅ 通过

测试内容：验证自动清理的触发条件

- 条件：sourceGroupId !== targetGroupId ✓

- 结果：触发自动清理 ✓
场景2：组内拖拽
- 条件：sourceGroupId === targetGroupId ✓

场景3：跨组拖拽但源组仍有标签
- 条件：sourceGroupId !== targetGroupId ✓
- 条件：updatedSourceGroup.tabs.length > 0 ✓
- 结果：不触发自动清理 ✓

🔧 技术实现详情

实现位置

主要修改文件：
- src/store/slices/tabSlice.ts (moveTabAndSync action)

使用的工具函数：
- shouldAutoDeleteAfterTabRemoval (检查删除条件)
- deleteGroup (删除标签组action)

协调的组件：
- SortableTabGroup (现有的自动删除逻辑)
- TabList/TabListDndKit (拖拽容器)

核心逻辑流程

1. 用户拖拽标签页到其他标签组
2. moveTabAndSync action执行标签页移动
3. 检查是否为跨组移动 (sourceGroupId !== targetGroupId)
4. 检查源标签组是否变空 (tabs.length === 0)
5. 使用工具函数验证删除条件 (未锁定等)
6. 从本地存储中移除空标签组
7. 延迟删除Redux状态中的标签组
8. 记录详细的操作日志
